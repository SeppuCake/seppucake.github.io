About the Project

A URL ending in .php is a direct indication that the website/blog uses SQL to deliver a lot of its data, and that we can execute queries directly by changing the URL. Now basically the data in the SQL tables is protected. However, when we send some rogue commands to the SQL server, it doesn't understand what to do, and returns an error. This is a clear indication that with proper coding, we can send queries that will make the database malfunction, and give us all the otherwise private data of its tables. This attack can be used to obtain confidential data like a list of username and passwords of all users on a website.

In this small project or practice, I tried to familiarize myself with Kali-Linux and the tools provided in the Operating System (OS) to understand the procedures in the most common attacks a website would face; SQL Injection Attacks. Through this method, I gained the knowledge in defining the aspects of secured environment for a website in a web developer's point of view. As demonstrations, I have used an open-source website that is provided by Acunetix and simple Google Hacking (by using Google Operators using 'inurl') in order to find the loopholes in a PHP written websites and conducted a manual SQL Injection.
